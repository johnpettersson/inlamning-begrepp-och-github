Inlämning: Begrepp och Github

Github Flow

Github flow är en metod för att arbeta tillsammans på github. I korthet så består det av nedanstående steg.

  1. Vi börjar med att skapa en ny branch så att vi kan göra ändringar utan att "förstöra" mainbranchen.
  2. Sedan implementerar vi någonting. Det kan t.ex. handla om någon ny feature eller en lösning på något issue.
  3. När ändringarna är commitade och pushade till branchen kan vi göra en pull request. I vår pull request kan vi lägga till vilka medlemmar av repot som ska godkänna den. Om det var en lösning till ett issue kan vi även länka till det. Medlemmarna i repot vi taggar kan då gå in och dubbelkolla vår nya kod och godkänna att den mergas in i huvudbranchen.
  4. Efter att en pullrequest är godkänd kan vi merga in den i huvudbranchen. Till sist kan vi ta bort branchen vi programmerat på.

Flexbox

Flexbox är ett sätt att hantera layout på med CSS. Istället för block/inline så sätter vi display: flex på en container. Den blir då en flex-container och alla containerns direkta barn blir flex-items Vi kan bland annat ändra olika CSS-attribut på containern för att:

  • ändra riktning på layouten med flex-direction
  • ändra hur containern distribuerar tillgängligt utrymme till sitt innehåll med flex-wrap
  • ändra hur containern lägger ut sitt innehåll både längs med sin riktning med justify-content och tvärs över med align-items.

På flex-items kan vi ändra hur mycket plats de ska ta upp i förhållande till varandra med flex-grow och ändra deras ordning med order.

DOM (Document Object Model)

DOM är en datastruktur som gör att vi kan manipulera vår HTML med javascript. Vi kan manipulera DOM genom flera olika properties och funktioner på document-objectet i javascript. Vi kan hitta element med getElementByID och querySelector. På varje element kan vi t.ex. också:
  • Ändra properties, t.ex. classList eller style för att förändra hur elementet ser ut
  • Förändra elementets ättlingar med appendChild, replaceChild, removeChild (eller ändra elementets innehåll direkt med innerHTML/innerText)
  • Lägga till eventlisteners som kör en viss funktion när elementet interageras med, t.ex. klickas på. Detta görs med addEventlistener

Fin kod

Fin kod handlar om att skriva på ett sätt som gör det lätt för andra att läsa och förstå din kod. Det underlättar givetvis även för dig själv när du återvänder till din kod efter x antal dagar/veckor/månader och kanske inte kom ihåg hur du tänkt.

Det är t.ex. viktigt att namnge variabler och funktioner så att man förstår deras syfte. Skriva kommentarer i sin kod, om man skriver någonting som kanske är lite mer advancerat eller vid första anblick inte är enkelt att förstå. Jag tycker också det är viktigt att vara enhetlig, bestäm dig för en kodstandard och följ den. Det kan t.ex. handla om att använda camelcase (stor bokstav på varje nytt ord i en variabel/funktion) ex. function reloadAllUsers() {} eller att alltid använda stora bokstäver till konstanter. Och arbetar man i grupp är också viktigt att komma överens om vilken kodstandard man ska följa.

Ramverk

Ramverk är färdiga bibliotek med kod som vi kan använda för att slippa behöva uppfinna hjulet varje gång man startar ett nytt projekt.

Exempel på ramverk som används inom webbutveckling för frontend är React, Vue och Angular

Bootstrap är ett ramverk för frontend som innehåller färdig CSS och JS som gör att vi kan göra responsiva layouts utan att egentligen behöva skriva någon CSS själva. Den färdiga javascripten i bootstrap gör att man kan lägga in komponenter utan att göra själva implementeringen av dem själv, t.ex. dropdowns eller modaldialoger.

Express.js är ett ramverk för backend som används med Node.js för att skapa webserverapplikationer.
